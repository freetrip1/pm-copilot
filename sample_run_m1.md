## ③ PRD 草稿

# 亲情服药提醒小程序 PRD
**一句话定位**：以子女远程代设、亲属录音强提醒及服药闭环反馈为核心的适老化服药提醒工具，解决独居老人漏服及子女担忧问题。

## 背景与问题
随着老龄化加剧，独居老人的服药依从性成为突出的家庭痛点。当前核心问题如下：
1. **记忆力衰退导致漏服**：老人日常生活中经常忘记按时吃药。
2. **视听障碍导致错过提醒**：常规手机文字闹钟看不清，默认短促提示音听不到，且操作不熟练。
3. **空间阻隔导致子女焦虑**：子女不在身边，无法亲自督促，对老人是否按时服药存在强烈的担忧与无力感。

**现有方案局限**：
- **Medisafe等国际App**：虽有时间表与家人共享，但本土化与适老化不足，缺乏亲属录音等情感化提醒，冷冰冰的机械音难以吸引老人注意。
- **智能药盒（如PillDrill）**：通过硬件传感器解决假确认痛点，但需额外购买，门槛高且普及难。
- **通用智能音箱**：音量大且支持语音设置，但缺乏复杂的周期性用药管理，且无法形成子女远程知晓的闭环反馈。
- **VP0吃药提醒组件**：提供了适老化基础方案，但在突破系统静音限制及防误触验证上仍有优化空间。

基于以上，我们需要一款**零硬件门槛、强情感连接、极简操作**的纯软件方案，填补市场空白。

## 目标用户与典型场景
### 目标用户
- **主要使用者（老人端）**：独居老人，存在记忆力衰退、视力/听力下降、智能手机操作不熟练等特征。
- **主要设置者（子女端）**：老人的子女或照护者，不在老人身边，关注老人健康，具备智能手机操作能力。

### 典型场景
1. **远程代设与情感录入**：子女在小程序中为老人录入药物名称和服药时间，并录制个性化语音（如“妈，该吃降压药了”），期望老人到点能听到熟悉的声音提醒。
2. **强提醒与按时服药**：到了服药时间，小程序自动突破常规音量限制播放大音量语音提醒，老人听到后拿出药物服用，不再漏服。
3. **极简确认反馈**：老人服药后，通过语音说出“吃完了”或点击屏幕上的超大按钮确认，期望子女能远程知晓自己已按时服药。
4. **漏服警报与干预**：超过设定时间老人未确认服药，小程序向子女发送漏服警报，期望子女能及时电话提醒避免健康风险。

## 产品目标与成功指标
### 产品目标
降低独居老人漏服率，提升子女安心感，验证“情感化强提醒+极简闭环”模式的有效性。

### 成功指标（可度量）
1. **提醒触达率**：提醒按时触发且老人端成功播报的比例 ≥ 95%（待确认：受限于系统后台限制，需以实际技术打点为准）。
2. **服药确认率**：触发提醒后，老人完成确认操作（语音/点击）的比例，目标 ≥ 待确认%（需上线后建立基线）。
3. **漏服干预率**：子女收到漏服警报后，在30分钟内跟进（如通过系统回拨电话）的比例 ≥ 60%。
4. **录音使用率**：设置服药计划的子女中，使用亲属录音功能的占比 ≥ 70%，验证情感化提醒的吸引力。

## 核心功能与需求清单
### P0（MVP核心闭环，必须完成）
1. **子女端-服药计划代设**：支持录入药物名称、单次剂量、服药时间（支持每日多次），并绑定老人微信/手机号。
2. **子女端-亲属录音上传**：支持子女录制个性化语音作为提醒铃声，或选择内置大音量提示音。
3. **老人端-强提醒播报**：到点触发全屏强提醒界面，自动大音量播放亲属录音/提示音，界面仅显示药物信息与超大确认按钮。
4. **老人端-极简确认操作**：提供占据屏幕1/3以上的超大“已服药”按钮，点击即完成反馈。
5. **子女端-漏服警报**：超过设定时间（如15分钟）老人未确认，子女微信收到服务通知（模板消息）提醒。

### P1（体验增强与防错，优先级高）
1. **老人端-语音口令确认**：支持老人对着手机说特定口令（如“吃完了”）完成确认，降低操作门槛。
2. **老人端-防误触机制**：点击确认后增加极简二次确认（如按钮变色并显示“再点一次确认”），或延迟3秒自动关闭，防止误触。
3. **子女端-复杂用药周期**：支持隔天一次、按周期循环、饭前/饭后等复杂规则的设置，老人端仅展示最终时间点。
4. **系统级静音突破**：申请系统级免打扰权限覆盖静音模式；若权限未获取，提供“电话语音外呼兜底”功能（子女授权下，漏服时自动拨打老人电话并播放TTS语音）。

### P2（探索性功能，视P0/P1数据排期）
1. **拍照打卡识别**：老人服药时拍照上传，子女端可查看照片核实（低成本替代智能药盒传感器）。
2. **NFC标签贴纸联动**：老人手机贴近药盒上的NFC贴纸即完成服药确认。
3. **多照护者协同**：支持多名子女绑定同一老人，共享服药状态。

## 范围与非目标
- **本期不做**：硬件药盒研发与销售；在线问诊与购药服务；非用药类的通用提醒（如喝水、运动）；多照护者协同管理（本期仅支持单一管理员）。
- **边界说明**：本期仅聚焦微信小程序生态，暂不开发独立原生APP。

## 价值假设与待验证问题
### 价值假设
1. 如果我们提供大音量、子女录音的个性化语音提醒，老人会因为听得清且感到亲切而显著降低漏服率。
2. 如果我们提供极简的确认操作（语音或超大按钮），老人会因为操作门槛低而愿意持续使用并主动反馈服药状态。
3. 如果我们提供子女远程代设置和漏服警报功能，子女会因为能跨时空掌握老人服药情况而获得安心感。

### 待验证问题
1. **后台播报限制**：微信小程序在锁屏或后台状态下，能否稳定触发语音播报？是否需要考虑系统级权限或原生APP？「待确认：需技术预研微信小程序后台播放能力及iOS/Android双端限制」
2. **静音模式突破**：如果老人手机处于静音模式或音量极小，如何确保语音提醒能被听到？「待确认：评估电话外呼兜底的成本与可行性」
3. **复杂规则必要性**：服药规则是否需要支持复杂场景（如隔天一次、按周期循环、饭前饭后等）？「待确认：MVP阶段是否先只支持每日固定时间，降低老人端认知负担」
4. **假确认防范**：如何防止老人误触确认按钮导致子女误以为已服药？是否有更可靠的服药验证机制？「待确认：引入AI语音识别特定口令、拍照打卡识别药盒，或结合NFC标签贴纸刮药盒，替代易误触的简单大按钮，解决假确认痛点」

## 风险与依赖
1. **技术风险（系统限制）**：微信小程序后台保活与锁屏下音频播放能力受限，是本产品的核心卡点。若无法稳定触发，需降级为“微信服务通知+电话外呼”组合方案。
2. **体验风险（静音漏接）**：老人习惯性将手机调至静音或震动，可能导致纯软件提醒完全失效，依赖子女兜底干预。
3. **合规与隐私风险**：亲属录音的存储与使用、老人健康数据（用药记录）的云端存储，需符合个人信息保护法及相关医疗数据合规要求。
4. **外部依赖**：若采用电话语音外呼兜底方案，需依赖第三方云通信供应商（如腾讯云/阿里云）的语音通知API稳定性。

## ⑤ 终版 PRD

# 亲情服药提醒小程序 PRD

**一句话定位**：以子女远程代设、亲属录音强提醒及服药闭环反馈为核心的适老化服药提醒工具，解决独居老人漏服及子女担忧问题。

## 背景与问题
随着老龄化加剧，独居老人的服药依从性成为突出的家庭痛点。当前核心问题如下：
1. **记忆力衰退导致漏服**：老人日常生活中经常忘记按时吃药。
2. **视听障碍导致错过提醒**：常规手机文字闹钟看不清，默认短促提示音听不到，且操作不熟练。
3. **空间阻隔导致子女焦虑**：子女不在身边，无法亲自督促，对老人是否按时服药存在强烈的担忧与无力感。

**现有方案局限**：
- **Medisafe等国际App**：虽有时间表与家人共享，但本土化与适老化不足，缺乏亲属录音等情感化提醒，冷冰冰的机械音难以吸引老人注意。
- **智能药盒（如PillDrill）**：通过硬件传感器解决假确认痛点，但需额外购买，门槛高且普及难。
- **通用智能音箱**：音量大且支持语音设置，但缺乏复杂的周期性用药管理，且无法形成子女远程知晓的闭环反馈。
- **VP0吃药提醒组件**：提供了适老化基础方案，但在突破系统静音限制及防误触验证上仍有优化空间。

基于以上，我们需要一款**零硬件门槛、强情感连接、极简操作**的纯软件方案，填补市场空白。

## 目标用户与典型场景
### 目标用户
- **主要使用者（老人端）**：独居老人，存在记忆力衰退、视力/听力下降、智能手机操作不熟练等特征。
- **主要设置者（子女端）**：老人的子女或照护者，不在老人身边，关注老人健康，具备智能手机操作能力。分为**管理员**（可编辑计划）与**观察者**（只接收警报）。

### 典型场景
1. **远程代设与情感录入**：子女在小程序中为老人录入药物名称和服药时间，并录制个性化语音（如“妈，该吃降压药了”），老人端通过验证码完成双向绑定授权。
2. **强提醒与按时服药**：到了服药时间，微信下发服务通知；若老人5分钟未点击，系统自动拨打老人电话（TTS+录音）进行强干预。老人接听或点击通知拉起小程序后，全屏播放亲属录音提醒。
3. **极简确认反馈**：老人服药后，点击屏幕上的超大按钮进行二次确认，消除铃声。即便弱网也先本地缓存给予成功反馈，后续静默同步。
4. **漏服警报与干预**：超过设定时间老人未确认，小程序向所有绑定子女（管理员及观察者）发送订阅消息；子女收到漏服警报后，在小程序内点击“已跟进”。

## 产品目标与成功指标
### 产品目标
降低独居老人漏服率，提升子女安心感，验证“电话外呼强干预+情感化提醒+极简闭环”模式的有效性。

### 成功指标（可度量）
1. **有效触达率**：
   - **电话外呼接通率**：漏服后触发外呼，老人接听电话的比例 ≥ 80%（作为强提醒的核心衡量指标）。
   - **小程序通知点击率**：微信服务通知下发后，老人点击拉起小程序的比例 ≥ 40%（辅助指标）。
2. **服药确认率**：触发提醒（接听电话或拉起小程序）后，老人完成确认操作（完成二次点击）的比例 ≥ 40%（MVP假设基线，上线后根据数据校准）。
3. **漏服干预率**：子女收到漏服警报后，在小程序内点击“已跟进”按钮的比例 ≥ 60%。
4. **录音使用率**：设置服药计划的子女中，使用亲属录音功能的占比 ≥ 70%，验证情感化提醒的吸引力。

## 核心功能与需求清单
### P0（MVP核心闭环，必须完成）
1. **子女端-服药计划代设**：
   - 支持录入药物名称（限20字符）、单次剂量（必须>0的数字，最多2位小数，最大值999）、服药时间（支持每日多次），并绑定老人微信/手机号。
   - 校验规则：同一时间（精确到分钟，前后相差5分钟内）设置多种药物需阻断保存并提示冲突；未完成老人端双向确认绑定前禁止保存计划。
2. **子女端-亲属录音上传**：
   - 支持子女录制个性化语音（最短1秒防空音，<1秒自动丢弃并提示；最长15秒防冗长，达到15秒自动截断保存），或选择内置提示音。
   - 异常处理：麦克风无权限/被占用时提示录制失败；上传超时（阈值30秒）提示网络异常并支持重试。
   - 合规要求：上传前必须勾选授权同意弹窗（声明录音仅用于设定的老人提醒，加密存储，关联服药计划删除时录音同步软删除）。
3. **老人端-强提醒播报**：
   - **核心干预路径**：到点通过微信下发服务通知 -> 若老人5分钟未点击拉起小程序，系统自动拨打老人电话播放TTS语音提醒（无论子女是否授权订阅消息，漏服即拨打老人电话）。
   - **小程序内播报**：老人点击通知或接听电话后打开小程序 -> 小程序内全屏播放亲属录音/提示音。录音循环播放，最大时长10分钟，10分钟后自动停止并标记为未确认。
   - **并发与异常处理**：若5分钟内有多个服药计划，合并为一次外呼/提醒；弱网/断网无法拉取云端录音时，降级播放小程序内置提示音；若老人正在微信语音通话，提示音播放不中断通话但以悬浮窗展示10秒并长震1次提示；老人杀进程或切后台，再次打开小程序时恢复未完成提醒的播放状态。
4. **老人端-极简确认与防误触操作**：
   - 提供占据屏幕1/3以上的超大“已服药”按钮。
   - **防误触机制（P0必须）**：点击一次后，铃声短暂暂停5秒，按钮变色并持续显示“再点一次确认服药”；若老人不再操作，铃声继续播放，但“再点一次”按钮保持常驻，直到老人完成二次点击，避免超时死循环。
   - **弱网处理**：点击确认后先写入本地缓存并给予成功UI反馈，网络恢复时静默同步；若超时未同步成功，老人端提示“反馈未送达，请检查网络”。服务端漏服判定容错：若在15分01秒收到端侧延迟同步的确认，覆盖漏服状态，并给子女补发“延迟确认”通知。
5. **子女端-漏服警报与电话外呼兜底**：
   - **触发逻辑**：超过设定时间15分钟老人未确认，子女收到微信订阅消息（需在绑定时引导勾选“总是保持以上选择”，若授权耗尽则降级发送短信）。
   - **电话外呼兜底（P0必须）**：漏服5分钟即向老人发起外呼。外呼TTS话术模板：“{老人称呼}，您好，现在是{时间}，您有{药物名称}需要服用，请及时服药”。外呼频次控制：每位老人每天最多外呼1次，避免骚扰。
   - **外呼异常流**：外呼重试前必须校验老人是否已确认，已确认则取消重试；老人接听但通话时长<3秒视为无效接听，计入“未确认”状态；无人接听间隔5分钟重试1次；若关机/占线/2次未接，则向子女发送短信通知“老人未接听提醒电话，请人工干预”。
6. **双向确认绑定与多角色管理**：
   - **双向验证**：子女输入老人手机号后，需向老人微信下发授权卡片或手机发送验证码，老人确认后方可绑定，防止恶意骚扰。
   - **多角色支持**：区分“管理员”（首个绑定者，可编辑计划）与“观察者”（后续绑定者，只接收漏服警报）。P0不支持多人同时编辑计划。
   - **解绑机制**：提供“申请解绑”线上流转功能，不直接展示对方隐私信息；观察者可自主解绑。

### P1（体验增强与防错，优先级高）
1. **老人端-语音口令确认**：
   - 支持老人对着手机说特定口令完成确认。因方言/口音容错率低，MVP暂缓上线。待预研微信原生同声传译插件或第三方ASR对方言的宽松匹配度后放入P1。
   - 异常流：识别失败、环境噪音过大或超时，自动弹出超大按钮确认界面降级引导。
2. **子女端-复杂用药周期**：支持隔天一次、按周期循环、饭前/饭后等复杂规则的设置，老人端仅展示最终时间点。
3. **老人端-音量引导**：小程序内提供醒目引导，提示老人手动调高系统媒体音量并关闭静音模式。

### P2（探索性功能，视P0/P1数据排期）
1. **拍照打卡识别**：老人服药时拍照上传，子女端可查看照片核实。
2. **NFC标签贴纸联动**：老人手机贴近药盒上的NFC贴纸即完成服药确认。

## 范围与非目标
- **本期不做**：硬件药盒研发与销售；在线问诊与购药服务；非用药类的通用提醒（如喝水、运动）；多人协同编辑服药计划。
- **边界说明**：本期仅聚焦微信小程序生态，暂不开发独立原生APP。核心强提醒受限于小程序后台能力，采用“微信服务通知+电话外呼强干预+点击拉起小程序”的降级架构实现。

## 价值假设与待验证问题
### 价值假设
1. 如果我们提供电话外呼强干预及大音量、子女录音的个性化语音提醒，老人会因为被有效触达且感到亲切而显著降低漏服率。
2. 如果我们提供极简的确认操作（超大按钮二次确认），并利用“必须点击才能停止铃声”的机制驱动，老人会因为操作门槛低且有消除噪音的动机而愿意主动反馈服药状态。
3. 如果我们提供子女远程代设置、多子女漏服警报功能，子女会因为能跨时空掌握老人服药情况而获得安心感。

### 待验证问题
1. **后台播报限制**：微信小程序在锁屏或后台状态下，能否稳定触发语音播报？「已确认：不可行。已降级为“微信服务通知->点击拉起小程序播放”路径」
2. **静音模式突破**：如果老人手机处于静音模式或音量极小，如何确保语音提醒能被听到？「已确认：纯软件无法突破系统静音。已将电话语音外呼作为核心强干预手段，漏服5分钟即触发」
3. **复杂规则必要性**：服药规则是否需要支持复杂场景？「待确认：MVP阶段先只支持每日固定时间，降低老人端认知负担与开发成本，P1再开放复杂周期」
4. **假确认防范**：如何防止老人误触确认按钮导致子女误以为已服药？「已解决：将防误触二次确认机制提升至P0，取消3秒超时，改为常驻提示二次点击」

## 风险与依赖
1. **技术风险（系统限制）**：微信小程序后台保活与锁屏下音频播放能力受限。已明确降级交互路径，核心卡点转为“老人是否会忽略微信服务通知不点击”，通过5分钟外呼兜底解决。
2. **体验风险（静音漏接）**：老人习惯性将手机调至静音或震动，可能导致小程序内提醒完全失效。依赖P0电话外呼兜底方案干预。
3. **合规与隐私风险**：亲属录音属生物特征数据，用药记录属健康数据。已增加授权同意弹窗，数据加密存储，并随服药计划删除同步软删除录音。
4. **外部依赖与成本**：电话外呼依赖第三方云通信供应商（SLA要求：接通延迟<3秒）。外呼按次计费，已增加频次控制（1天/人1次）防范成本爆炸。微信订阅消息依赖用户授权，需强引导勾选“总是保持以上选择”。
